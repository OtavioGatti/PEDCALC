Você é um desenvolvedor Full Stack Sênior. Vamos construir o PedCalc, um PWA (Progressive Web App) focado em ser uma calculadora pediátrica offline-first para uso médico em plantões e enfermarias. O objetivo principal do app é calcular a posologia correta em mL a partir do peso e idade da criança, validar a segurança do medicamento para a idade informada e gerar uma receita de "cópia rápida".Stack TecnológicaCMS (Painel Administrativo): Notion API (onde o usuário médico alimentará a tabela de medicamentos).Backend: Node.js/TypeScript hospedado na Render (responsável pelo script de sincronização).Banco de Dados: Supabase (PostgreSQL) com RLS (Row Level Security) ativo.Frontend: React (Vite) ou Vue.js com Tailwind CSS, hospedado na Vercel, configurado como PWA.🛠️ ETAPA 1: O Backend (Sincronizador Notion ➡️ Supabase)Crie um script em Node.js (TypeScript) para rodar na Render. Ele deve expor uma rota GET /api/sync protegida por um token simples de API.1. Schema da Tabela no Supabase (medicamentos)Gere a estrutura da tabela SQL considerando:id: uuid (Primary Key)nome: textprincipio_ativo: textconcentracao_valor: numericconcentracao_unidade: text (ex: 'mg/mL')idade_min_valor: numeric (nullable)idade_min_unidade: text (ex: 'Dias', 'Meses', 'Anos')idade_max_valor: numeric (nullable)idade_max_unidade: text (ex: 'Dias', 'Meses', 'Anos')alerta_restricao: text (nullable)dose_alvo_mg_kg_dia: numericfracionamento_vezes_dia: numeric (ex: 3 para 8/8h, 4 para 6/6h)texto_prescricao_padrao: text2. Lógica do Script de SincronizaçãoFazer uma requisição para a API do Notion puxando o banco de dados especificado.Mapear e tratar as propriedades do Notion (sanitizar para garantir que números sejam salvos como numeric).Executar um .upsert() no Supabase usando o nome do medicamento ou o ID do Notion como chave de conflito, garantindo que registros novos sejam criados e antigos sejam atualizados.🧠 ETAPA 2: O Motor de Cálculo e Validação (Frontend)O frontend operará de forma offline-first. Ao carregar o app com internet, ele baixa a tabela inteira do Supabase e salva no LocalStorage ou IndexedDB. Os cálculos serão feitos localmente no client-side.1. Lógica de Conversão de IdadeCrie uma função utilitária que converta qualquer idade fornecida pelo usuário e as idades limites do medicamento para uma unidade comum (ex: Dias) para fazer a validação lógica:Se Idade_Paciente < Idade_Minima OU Idade_Paciente > Idade_Maxima: Ativar estado de erro/alerta e injetar na UI o texto do campo alerta_restricao.2. Equações do Motor de CálculoAplique a seguinte lógica matemática rigorosa para o cálculo da dose:$$\text{Dose Absoluta Diária (mg)} = \text{Peso (kg)} \times \text{Dose Alvo (mg/kg/dia)}$$$$\text{Dose por Tomada (mg)} = \frac{\text{Dose Absoluta Diária (mg)}}{\text{Fracionamento (Vezes/Dia)}}$$$$\text{Volume por Tomada (mL)} = \frac{\text{Dose por Tomada (mg)}}{\text{Concentração Valor (mg/mL)}}$$🎨 ETAPA 3: Interface do Usuário (UI/UX) & Paleta de CoresA interface deve ser extremamente limpa, de alto contraste, otimizada para dispositivos móveis (Mobile-First).Diretrizes de Cores (Tailwind)Cor Primária/Destaque: #5ADB91 (Verde Menta).
